# 标题判断规则

本文档定义 `office-copy-as-markdown` 中标题识别的最终规则。

## 总流程

标题识别分两步：

1. 先检查当前选中片段里是否存在语义标题
2. 只有在不存在语义标题时，才启用候选标题推断

如果当前选中片段里已经存在语义标题，则整个片段禁用样式推断，只按语义标题输出。

## 语义标题优先规则

以下任一情况都视为语义标题：

- HTML 节点是 `<h1>` 到 `<h6>`
- 节点样式中包含 `mso-style-name: Heading N`
- 节点类名中包含 `HeadingN`

映射规则：

- `Heading 1` 或 `<h1>` -> `#`
- `Heading 2` 或 `<h2>` -> `##`
- `Heading 3` 或 `<h3>` -> `###`
- 依此类推到 `Heading 6`

## 基础候选标题规则

只有在当前选中片段中不存在语义标题时，才会开始筛选基础候选标题。

基础候选标题必须同时满足：

- 独占一行
- 非列表项
- 非表格单元格
- 非引用
- 非代码块
- 文本非空
- 文本长度不超过 `30` 个字符
- 可提取到有效字号
- 文本末尾不是 `。 . ， , ； ;`

## 有效候选标题规则

基础候选标题并不会直接转成 Markdown 标题，还需要继续判断是否满足字号优势，或满足“粗体且字号不小于正文”的条件。

正文字号基线：

- 在当前选中片段内统计可提取到的字号
- 取“最常见字号”作为正文字号基线

只有满足以下任一条件的基础候选标题，才会成为有效候选标题：

- 1. 字号至少比正文大 `2pt`，或字号至少比正文大 `15%`
- 2. 文本为粗体，且字号不小于正文字号基线

补充说明：

- 粗体只影响“是否成为有效候选标题”，不单独决定标题级别
- 粗体规则只在“基础候选标题”范围内生效，不会绕过前置过滤条件

## 标题级别映射规则

标题级别映射只针对有效候选标题执行，并且只在当前选中片段中不存在语义标题时启用。

当前候选标题推断的支持级别上限由独立配置项控制；当前默认值为 `4`。
当当前有效候选标题的字号档位少于该上限时，起始映射级别也由独立配置项控制；当前默认值为 `2`，即从 `##` 开始映射。

补充说明：

- 有效候选标题仍然只按字号分档映射 Markdown 标题级别
- 粗体不会改变语义标题优先级，也不会引入单独的“粗体标题档位”
- 与正文同字号的粗体候选标题，可以进入当前有效候选标题中的最低有效字号档位
- 若当前有效候选标题的字号档位少于“最大支持级别”配置值，则从“稀疏映射起始级别”配置值开始映射；当前默认行为是从 `##` 开始映射

规则：

1. 取所有有效候选标题的字号
2. 去重
3. 按字号从大到小排序
4. 若字号档位数小于“最大支持级别”配置值，则从“稀疏映射起始级别”配置值开始映射；否则从 `#` 开始映射
5. 按排序结果映射 Markdown 标题级别

映射关系：

- 第一档 -> `#`
- 第二档 -> `##`
- 第三档 -> `###`
- 第四档 -> `####`
- 第五档及以下 -> 不输出标题

## 字号提取规则

字号提取顺序：

1. 优先读取块节点自身 `style` 中的 `font-size`
2. 若块节点没有字号，则继续查找其主要文本子节点中的 `font-size`

支持单位：

- `pt`
- `px`

内部统一换算为 `pt` 再参与比较。

## 典型例子

### 有语义标题

如果当前选中片段里包含：

- `Heading 1`
- 普通大字号短句

则只按 `Heading 1` 输出 `#`，普通大字号短句不会再参与候选标题推断。

### 无语义标题，按字号分档

如果当前选中片段没有语义标题，并且统计结果为：

- 正文最常见字号：`12pt`
- 候选标题字号：`18pt`、`16pt`、`14pt`、`12pt + 粗体`

则输出：

- `18pt` -> `#`
- `16pt` -> `##`
- `14pt` -> `###`
- `12pt + 粗体` -> `####`

### 无语义标题，但字号档位少于最大支持级别

如果当前选中片段没有语义标题，并且统计结果为：

- 正文最常见字号：`12pt`
- 候选标题字号：`12pt + 粗体`
- 最大支持级别配置：`4`
- 稀疏映射起始级别配置：`2`

则输出：

- `12pt + 粗体` -> `##`

### 无语义标题，但不会成为有效候选标题

如果当前选中片段没有语义标题，并且正文字号基线为 `12pt`，则以下内容都不会成为有效候选标题：

- `12pt` 但非粗体
- `11pt + 粗体`

### 不会进入候选标题的情况

以下内容即使单独一行，也不会参与标题推断：

- 超过 `30` 个字符的文本
- 末尾是 `。 . ， , ； ;` 的文本
- 列表项
- 表格单元格
- 引用块中的文本
- 代码块中的文本

即使这些内容本身是粗体，也不会因为粗体而绕过基础候选标题过滤。

## 已知限制

- 统计范围只限“当前选中片段”，不会扫描整篇文档
- 候选标题推断的级别上限只在“无语义标题”的前提下生效；一旦存在语义标题，整个片段仍然只按语义标题输出
- 当前候选标题推断最多映射到“最大支持级别”配置值所允许的 Markdown 标题层级；当前默认值为 `4`
- 当字号档位少于“最大支持级别”配置值时，推断结果会从“稀疏映射起始级别”配置值开始映射；当前默认行为是不占用 `#`，而是从 `##` 开始映射
- 如果当前片段只包含标题而没有足够的正文样本，正文字号基线可能不够稳定
- 粗体规则只影响有效候选标题判定；列表项、表格单元格、引用、代码块、超长文本、末尾带禁用标点的文本，仍不能因为加粗而变成标题
- OneNote 和 Word 复制出的 HTML 结构并不完全一致，复杂样式仍可能需要后续微调
