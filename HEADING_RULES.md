# 标题判断规则

本文档定义 `office-copy-as-markdown` 中标题识别的最终规则。

## 总流程

标题识别分两步：

1. 先检查当前选中片段里是否存在语义标题
2. 只有在不存在语义标题时，才启用候选标题推断

如果当前选中片段里已经存在语义标题，则整个片段禁用样式推断，只按语义标题输出。

## 语义标题优先规则

以下任一情况都视为语义标题：

- HTML 节点是 `<h1>` 到 `<h6>`
- 节点样式中包含 `mso-style-name: Heading N`
- 节点类名中包含 `HeadingN`

映射规则：

- `Heading 1` 或 `<h1>` -> `#`
- `Heading 2` 或 `<h2>` -> `##`
- `Heading 3` 或 `<h3>` -> `###`
- 依此类推到 `Heading 6`

## 基础候选标题规则

只有在当前选中片段中不存在语义标题时，才会开始筛选基础候选标题。

基础候选标题必须同时满足：

- 独占一行
- 非列表项
- 非表格单元格
- 非引用
- 非代码块
- 文本非空
- 文本长度不超过 `20` 个字符
- 可提取到有效字号
- 文本末尾不是 `。 . ， , ； ;`

## 有效候选标题规则

基础候选标题并不会直接转成 Markdown 标题，还需要继续判断字号是否明显大于正文。

正文字号基线：

- 在当前选中片段内统计可提取到的字号
- 取“最常见字号”作为正文字号基线

只有满足以下条件的基础候选标题，才会成为有效候选标题：

- 字号至少比正文大 `2pt`
- 或字号至少比正文大 `15%`

## 标题级别映射规则

标题级别映射只针对有效候选标题执行。

规则：

1. 取所有有效候选标题的字号
2. 去重
3. 按字号从大到小排序
4. 按排序结果映射 Markdown 标题级别

映射关系：

- 第一档 -> `#`
- 第二档 -> `##`
- 第三档 -> `###`
- 第四档及以下 -> 不输出标题

## 字号提取规则

字号提取顺序：

1. 优先读取块节点自身 `style` 中的 `font-size`
2. 若块节点没有字号，则继续查找其主要文本子节点中的 `font-size`

支持单位：

- `pt`
- `px`

内部统一换算为 `pt` 再参与比较。

## 典型例子

### 有语义标题

如果当前选中片段里包含：

- `Heading 1`
- 普通大字号短句

则只按 `Heading 1` 输出 `#`，普通大字号短句不会再参与候选标题推断。

### 无语义标题，按字号分档

如果当前选中片段没有语义标题，并且统计结果为：

- 正文最常见字号：`12pt`
- 候选标题字号：`18pt`、`16pt`、`14pt`

则输出：

- `18pt` -> `#`
- `16pt` -> `##`
- `14pt` -> `###`

### 不会进入候选标题的情况

以下内容即使单独一行，也不会参与标题推断：

- 超过 `20` 个字符的文本
- 末尾是 `。 . ， , ； ;` 的文本
- 列表项
- 表格单元格
- 引用块中的文本
- 代码块中的文本

## 已知限制

- 统计范围只限“当前选中片段”，不会扫描整篇文档
- 无语义标题推断最多只映射到 3 级 Markdown 标题
- 如果当前片段只包含标题而没有足够的正文样本，正文字号基线可能不够稳定
- OneNote 和 Word 复制出的 HTML 结构并不完全一致，复杂样式仍可能需要后续微调
