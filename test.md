**摸鱼卡片（manycards.cn）产品规格**

**阶段**：MVP**目标用户**：UGC创作者+碎片化自学学习者**核心价值**：支持用户创建官方模板卡片、组织合集与章节、制定学习计划、个性化笔记及收藏

# 核心概念

| **名称** | **描述** |
| --- | --- |
| **Member** | 平台统一用户角色，既可以创建卡片/合集（创作者），也可以学习/收藏（学习者） |
| **卡片 Card** | 基于官方模板创建（单词、句子、成语、古诗、考试题等），用户无法自定义模板，但可以填写内容，同一张卡片只能属于一个合集 |
| **合集 CardSet** | 卡片的集合，支持章节（最多两层，类似书本→章节），同一合集允许多种卡片模板，卡片模板是官方定义不可自定义 |
| **章节** | 合集中的子合集，层级仅两层，不可嵌套更多 |
| **笔记 Note** | 用户对卡片添加的个性化文本或手写笔记，笔记绑定具体合集和卡片，即set_id+card_id |
| **学习计划 Plan** | 用户针对合集/章节制定每日学习任务及打卡记录，针对同一合集，用户可以制定多个计划，深度学习或者快速复习 |
| **收藏 Favorite** | 收藏合集或章节，方便后续学习或试学 |
| **卡片模板** | 官方定义，不可自定义，合集可混合不同模板卡片 |
| **访问/权限** | 根据合集可见性（私有/免费/付费）和用户身份进行访问控制，MVP阶段付费暂不实现结算，仅做标记与权限判断 |

# 用户路径

同一用户可以是学习者，也可以是创作者。

## 2.1 学习者路径

1. **发现**：
  - 浏览合集库→查看合集信息（封面、简介、是否公开/免费）
2. **获取**：
  - 免费合集→收藏或添加到学习计划
  - 付费合集→收藏（购买逻辑暂不实现）
3. **个性化**：
  - 对卡片添加笔记（文本/手写）
  - 跟踪完成情况（打卡）
4. **坚持**：
  - 根据学习计划完成每日任务
  - 系统记录完成量和用时
5. **分享**：
  - 分享合集、章节或卡片链接
  - 链接附带来源信息，受访问权限控制

## 2.2 创作者路径

1. **创建合集**：
  - 新建合集→可选创建章节（最多一层子合集）
  - 添加卡片到合集（从别的合集添加），或者创建卡片→支持多种官方模板卡片混合
2. **创建卡片**：
  - 选择官方模板→填写内容→保存卡片
3. **管理卡片**：
  - 同一张卡片只能属于一个合集或章节
  - 支持排序及编辑内容
  - 付费合集可部分可见
4. **发布/分享**：
  - 设置合集可见性（私有/免费）
  - 分享合集或章节链接

# 权限与规则

1. **访问控制**
  - 用户只能访问自己创建的合集/章节或已获得权限的合集
  - 付费合集未购买用户无法访问
2. **卡片添加规则**
  - 卡片只能属于一个合集/章节
  - 同一合集内允许多种官方模板卡片
3. **笔记与收藏**
  - 仅属于个人，可附带来源合集信息
  - 收藏/笔记不影响原卡片权限
4. **学习计划**
  - 用户自定义计划周期和每日任务量
  - 打卡记录仅对自己可见

# 开发重点（MVP 阶段）

- 用户登录注册
- 统一用户角色：Member 即创作者 + 学习者
- 章节支持：合集 → 可选子合集 → 卡片
- 卡片模板限制：只能使用官方定义模板
- 单合集引用：同一卡片只能属于一个合集或章节
- 权限控制：私有/免费/付费标记，访问权限检查
- 浏览和探索
- 个性化笔记和收藏：记录用户操作，不影响卡片访问
- 分享功能：带来源信息，遵循访问权限

# MVP 阶段不做

- 完整付费结算和购买流程
- 子子合集（超过两层层级）
- 智能推荐学习计划或自动提醒
