# 全文完整性校验与补回规则

## 目标

1. 最终输出必须是 Markdown。
2. Markdown 的可见文本必须尽量完整覆盖源纯文本。
3. 当结构化转换与内容完整冲突时，优先保证内容完整。
4. 不允许为了补回内容而输出原始 HTML。

## 基线规则

1. 以源纯文本作为完整性基线。
2. 以转换后的 Markdown 作为待校验结果。
3. 比较只关注可见文本，不要求排版与 Office 完全一致。
4. 标题只能来自原始结构判断，补回阶段不得新增标题。

## 源文本拆分规则

1. 统一换行符。
2. 按行拆分。
3. 跳过空白行。
4. 每行按顺序识别为：
   - 任务列表
   - 有序列表
   - 无序列表
   - 普通段落
5. 识别出的行类型必须在补回时继续保留。

## Markdown 比较规则

1. 图片按可见替代文本比较。
2. 链接按链接文本比较。
3. 加粗、斜体、删除线、代码标记不参与比较。
4. 标题、引用、列表前缀在比较前剥离。
5. 空结果行不参与比较。

## 归一化规则

1. 连续空白折叠为一个空格。
2. `Tab` 视为空格。
3. `→` 统一为 `->`。
4. 全角 `＋` 统一为 `+`。
5. 中英文之间的可选空格不作为差异。
6. 中文与中文之间的多余空格不作为差异。
7. `/`、`:`、`：`、`+` 周围的格式性空格不作为差异。

## 表格规则

1. Markdown 表格与纯文本表格按单元格内容比较。
2. 表格分隔行不参与比较。
3. 只要单元格文本一致，就视为已覆盖。
4. 不因为表格语法与纯文本分隔方式不同而判定缺失。

## 完整性判定规则

1. 采用顺序匹配。
2. 每条源文本只在当前位置之后的 Markdown 中寻找第一个匹配项。
3. 未匹配到的源文本视为缺失内容。
4. 只有缺失内容数为 0，才视为完整。

## 补回规则

1. 缺失内容必须补回到 Markdown。
2. 补回时保留原行类型：
   - 任务列表补回为任务列表
   - 有序列表补回为有序列表
   - 无序列表补回为无序列表
   - 普通文本补回为段落
3. 优先插入到前一个已匹配内容之后。
4. 若前方无锚点，则插入到后一个已匹配内容之前。
5. 若前后都无锚点，则追加到文末。
6. 补回后允许最小空行调整，但不得制造多余连续空白段。

## 回退规则

1. 若结构化 Markdown 校验失败，先执行补回。
2. 若补回后仍不完整，则退化为保守 Markdown。
3. 保守 Markdown 只要求内容完整，不要求结构最优。
4. 保守 Markdown 仍须再次通过完整性校验。

## 禁止规则

1. 不得因校验失败而静默丢弃内容。
2. 不得在补回阶段把普通文本升级为标题。
3. 不得为了通过校验而重复插入已存在内容。
4. 不得因为表格、链接、图片或中英混排的表示差异而误判缺失。

## 一句话原则

先生成尽量好的 Markdown，再按源纯文本逐行核对；凡是 Markdown 没覆盖到的内容，都必须以 Markdown 形式补回，直到结果完整。
